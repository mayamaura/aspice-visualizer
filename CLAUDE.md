## プロジェクト概要

- **アプリ名:** Automotive SPICE 4.0 Process Visualizer
- **目的:** ASPICE 4.0 の全33プロセスとその関係性（プロセス間フロー・基本プラクティス・プロセス成果・アウトプット情報項目）を複数ビューで可視化する読み取り専用Webアプリ
- **対象ユーザー:** 自動車業界のASPICEアセスメント・開発プロセス整備担当者

## ブランチ戦略

```
main          ← 安定版。直接コミット禁止
feat/*        ← 機能追加（例: feat/dagre-layout）
fix/*         ← バグ修正
docs/*        ← ドキュメントのみの変更
```

- 作業は必ずブランチを切ってから開始する
- `main` へのマージは `--no-ff`（マージコミット作成）

---

## コーディング規約

- TypeScript strict モード維持（型エラーゼロ）
- コメントは原則不要。WHYが非自明な場合のみ1行
- 新規ファイル作成より既存ファイル編集を優先
- バックエンド・外部APIを追加しない（静的データのみ）

---

## データ設計原則

- すべてのテキストフィールドは `BilingualText { en: string; ja: string }` 型で保持
- 言語切替は `t(text, lang)` ユーティリティで行う（ハードコーディング禁止）
- プロセスデータは `src/data/processes/<group>.ts` に分離して管理
- 新プロセスグループは専用ファイルを作成し `src/data/index.ts` でエクスポート

## 現在の実装状況

### 実装済みビュー
| ビュー | 説明 |
|---|---|
| プロセスマップ | 8グループ・33プロセスをタイル表示。グループフィルター付き。クリックで詳細パネル（目的・成果・BP・情報項目）を表示 |
| リレーションシップグラフ | プロセス間の情報フローをグラフ表示。プロセスレベル（グループフィルター付き）とBP/情報項目レベル（エッジ種別フィルター付き）を切替可能 |

### 実装済みフィルター機能
| フィルター | 適用箇所 |
|---|---|
| グループフィルター（GroupFilterBar） | プロセスマップ / リレーショングラフ（プロセスレベル）— 8グループ個別ON/OFF、最低1グループ必須 |
| エッジ種別フィルター（EdgeTypeFilterBar） | リレーショングラフ（BP/情報項目レベル）— supports/produces/input 個別ON/OFF、supports OFFでOutcomeノードも非表示 |

### 網羅プロセス（33プロセス）

|---|---|
| SYS | SYS.1〜SYS.5 |
| SWE | SWE.1〜SWE.6 |
| HWE | HWE.1〜HWE.5 |
| MAN | MAN.3, MAN.5, MAN.6 |
| SUP | SUP.1, SUP.2, SUP.4, SUP.7, SUP.8, SUP.9, SUP.10 |
| ACQ | ACQ.3, ACQ.4, ACQ.13, ACQ.14 |
| SPL | SPL.1, SPL.2 |
| REU | REU.2 |
