# 要求仕様書

**プロジェクト名:** Automotive SPICE 4.0 Process Visualizer  
**バージョン:** 1.4  
**最終更新:** 2026-05-20  

---

## 1. プロジェクト概要

### 1.1 目的

Automotive SPICE (ASPICE) 4.0 で定義されたすべてのプロセスと、それらの関係性（プロセス間フロー・基本プラクティス・プロセス成果・アウトプット情報項目の依存関係）を複数のビューで見える化する、**読み取り専用のインタラクティブWebアプリケーション**を提供する。

### 1.2 対象ユーザー

- 自動車業界でASPICEアセスメント・開発プロセス整備に携わるエンジニア、品質担当者
- ASPICE 4.0 の構造・プロセス間関係を学習する人

### 1.3 スコープ

- ASPICE 4.0（PAM v4.0 + CS-PAM v2.0）の全プロセス（38プロセス、12グループ）を網羅
- 読み取り専用（編集・入力機能なし）
- スタンドアロンWebアプリ（外部API不使用、静的JSONデータのみ）

## 2. 機能要件

### 2.1 プロセスマップビュー（FR-1）

| ID | 要件 |
|---|---|
| FR-1-1 | SYS / SWE / HWE / VAL / MLE / MAN / SUP / PIM / ACQ / SPL / REU / SEC の12グループをグループごとに色分けしてタイル表示する |
| FR-1-2 | 各プロセスカードにはプロセスID（例: SWE.1）とプロセス名を表示する |
| FR-1-3 | プロセスカードをクリックすると右側に詳細パネルをスライド表示する |
| FR-1-4 | 詳細パネルには「プロセス目的」「プロセス成果」「基本プラクティス」「アウトプット情報項目」を表示する |
| FR-1-5 | 基本プラクティスには、支援するプロセス成果ID（oc.N形式）・注記（notes）を表示する |
| FR-1-6 | 詳細パネルの各セクションは折りたたみ可能とする |
| FR-1-7 | 詳細パネルはクローズボタンで閉じられる |
| FR-1-8 | グループフィルターバーで表示するプロセスグループを絞り込める（最低1グループ選択必須） |
| FR-1-9 | フィルターで非表示になったグループのプロセスが詳細パネルで選択中の場合、パネルを自動的に閉じる |

### 2.2 リレーションシップグラフビュー（FR-2）

| ID | 要件 |
|---|---|
| FR-2-1 | プロセスレベルでは、ASPICE 4.0 のプロセスアーキテクチャ図に準拠した固定レイアウトでプロセスノードを表示する。各プロセス群はグループコンテナ（枠）で囲んで視覚的にグルーピングする |
| FR-2-1a | プロセス群コンテナ内のプロセスノード配置はアーキテクチャ図の空間的位置関係（左列=SUP、上段中央=SYS/VAL、右列=MAN、中段=SWE/HWE、下段=ACQ/SEC/MLE/PIM/REU/SPL）に従う |
| FR-2-2 | ASPICE 4.0 にはプロセス間の明示的な情報フローが存在しないため、プロセスレベルではエッジを描画しない |
| FR-2-3 | プロセスレベルでプロセスノードをクリックすると、そのプロセスをBP/情報項目レベルに展開する |
| FR-2-4 | BP/情報項目レベルでは、プロセス・プロセス成果・基本プラクティス・情報項目の4種ノードを表示する |
| FR-2-5 | BP/情報項目レベルのエッジ種別: BP→成果（supports/達成）/ 成果→情報項目（produces/生成） |
| FR-2-6 | レベル切替（プロセスレベル / BP/情報項目レベル / 情報項目起点）をツールバーボタンで行う |
| FR-2-7 | グラフはズーム・パン操作が可能 |
| FR-2-8 | ミニマップを表示する |
| FR-2-9 | BP/情報項目レベルでは「戻る」ボタンでプロセスレベルに戻れる |
| FR-2-10 | プロセスレベルにグループフィルターバーを表示し、表示するプロセスグループを絞り込める（最低1グループ必須） |
| FR-2-11 | BP/情報項目レベルにエッジ種別フィルターバーを表示し、supports / produces のエッジを個別にON/OFFできる（最低1種別必須） |
| FR-2-12 | 情報項目起点レベルでは、全プロセスが出力する情報項目ノードを一覧表示する |
| FR-2-13 | 情報項目ノードをクリックすると、その情報項目を生成するすべてのプロセス成果とプロセスを左から右（プロセス → 成果 → 情報項目）の形式で展開する |
| FR-2-14 | 情報項目起点グラフでは「戻る」ボタンで情報項目一覧に戻れる |
| FR-2-15 | 情報項目起点グラフでもノードホバー時に接続ノード・エッジを強調表示する |
| FR-2-16 | 情報項目起点グラフ表示中は、選択中の情報項目の詳細（ID・名称・説明・特性）を右側パネルに表示する。Xボタンで閉じると情報項目一覧に戻る |

### 2.3 言語切替（FR-3）

| ID | 要件 |
|---|---|
| FR-3-1 | 英語（EN）と日本語（JA）の切替ボタンをヘッダーに常時表示する |
| FR-3-2 | 切替は即時反映し、すべてのUIテキスト・データラベルに適用される |
| FR-3-3 | プロセス名・目的・成果・BP説明・情報項目名・特性はすべて EN/JA 両言語で保持する |

### 2.4 ナビゲーション（FR-4）

| ID | 要件 |
|---|---|
| FR-4-1 | ヘッダーのタブでビュー（プロセスマップ / リレーションシップグラフ）を切替できる |
| FR-4-2 | 初期表示はプロセスマップビューとする |

## 3. 非機能要件

| ID | 分類 | 要件 |
|---|---|---|
| NFR-1 | パフォーマンス | 初回ロード 3秒以内（ローカル環境）、ビュー切替 1秒以内 |
| NFR-2 | 保守性 | プロセスデータは `aspice_models.json` 単一ファイルで管理し、ファイルを差し替えるだけで内容を更新できる。スキーマ変更時は `aspiceRaw.ts` / `aspiceLoader.ts` のみ修正すればよい設計とする |
| NFR-3 | 拡張性 | 新ビュー（Vモデルビュー、成果物フロービューなど）をコンポーネント追加で対応できる構造とする |
| NFR-4 | 型安全性 | TypeScript strict モードで実装し、型エラーゼロを維持する |
| NFR-5 | ブラウザ対応 | Chrome / Edge 最新版で動作すること |

## 4. データ要件

### 4.1 網羅プロセス一覧

正確なプロセスIDは `src/data/aspice_models.json` を参照。以下はグループ別の件数。

| グループ | 由来 | プロセス数 |
|---|---|---|
| ACQ（Acquisition） | PAM | 2 |
| SPL（Supply） | PAM | 1 |
| SYS（System Engineering） | PAM | 5 |
| SWE（Software Engineering） | PAM | 6 |
| VAL（Validation） | PAM | 1 |
| MLE（Machine Learning Engineering） | PAM | 4 |
| HWE（Hardware Engineering） | PAM | 4 |
| SUP（Supporting） | PAM | 5 |
| MAN（Management） | PAM | 4 |
| PIM（Process Improvement） | PAM | 1 |
| REU（Reuse） | PAM | 1 |
| SEC（Security） | CS-PAM | 4 |
| **合計** | | **38** |

### 4.2 各プロセスで保持するデータ

- プロセスID、プロセス名（EN/JA）、グループ
- プロセス目的（EN/JA）
- プロセス成果（id: number、説明 EN/JA）
- 基本プラクティス（ID、名称 EN/JA、説明 EN/JA、支援するプロセス成果ID（number[]）、注記 EN/JA）
- アウトプット情報項目参照（情報項目ID、対応プロセス成果ID）
- 情報項目マスタ（ID、名称 EN/JA、特性リスト（type: bullet/category/note）） ← `information_items` として分離管理

---

## 5. 制約事項

- 読み取り専用。ユーザーによるデータ入力・編集機能は提供しない
- バックエンド・データベース不使用。すべてのデータは静的TypeScriptファイルとして管理
- ケイパビリティレベル入力・アセスメント機能は現バージョンのスコープ外
